# 紧急修复方案 - 404错误

## 问题诊断

你访问 https://hsq66.github.io/yuhang/ 时出现404错误。

## 最可能的原因

GitHub Pages配置中的 **Folder** 选项选错了！

## 立即检查

1. 访问：https://github.com/hsq66/yuhang/settings/pages

2. 查看 "Branch" 部分的配置：
   - 第一个下拉菜单应该是：**main** ✅
   - 第二个下拉菜单应该是：**/docs** ⚠️

**关键问题**：第二个下拉菜单是 `/docs` 还是 `/ (root)`？

## 解决方案

### 如果第二个下拉菜单是 `/ (root)`：

这就是问题所在！GitHub Pages在根目录找不到index.html文件。

**修复步骤**：
1. 将第二个下拉菜单改为 **/docs**
2. 点击 **Save**
3. 等待2分钟
4. 刷新 https://hsq66.github.io/yuhang/

### 如果第二个下拉菜单已经是 `/docs`：

可能是以下原因之一：

#### 方案A：等待更长时间
首次部署可能需要5-10分钟。请再等待5分钟后刷新。

#### 方案B：检查部署状态
访问：https://github.com/hsq66/yuhang/deployments

查看部署状态是否成功。

#### 方案C：强制重新部署

执行以下命令触发重新部署：

```bash
# 创建一个空提交
git commit --allow-empty -m "触发GitHub Pages重新部署"
git push origin main
```

然后等待2-3分钟。

## 快速测试

尝试访问以下URL，告诉我哪些能访问：

1. https://hsq66.github.io/yuhang/
2. https://hsq66.github.io/yuhang/index.html
3. https://hsq66.github.io/yuhang/css/style.css
4. https://hsq66.github.io/yuhang/images/one.jpg

如果2、3、4能访问，但1不能，说明是index.html的路由问题。

## 终极解决方案：重新构建

如果上述方案都不行，执行以下步骤：

### 步骤1：清理并重新构建

```bash
# 删除docs目录
rmdir /s /q docs

# 重新构建
cd yuhang-hugo
hugo --destination ../docs --cleanDestinationDir
cd ..
```

### 步骤2：确认文件存在

```bash
# 检查index.html是否存在
dir docs\index.html

# 应该看到文件大小约16KB
```

### 步骤3：提交并推送

```bash
git add docs/
git commit -m "重新构建网站"
git push origin main
```

### 步骤4：等待部署

等待3-5分钟后访问 https://hsq66.github.io/yuhang/
